Constraint Layout 初見

2016-06-18
#Android #ConstraintLayout #Codelabs

什麼是 ConstraintLayout

  • 訂定 View／layout 之間的 constraint, 用以適應不同 size 的螢幕
  • 簡化 layout hierarchies，終於不用再 RelativeLayout 包 N 層了！
  • 增加 layout performance
  • 還不是抄 iOS 的 Autolayout

要怎麼開始使用呢

  1. 下載 Android Studio 2.2（目前還在 preview 3）
  2. 在 build.gradle 加上
    compile 'com.android.support.constraint:constraint-layout:1.0.0-alpha3'

教學

  • Codelabs-Constraint Layout
    • 基本上跑完這篇教學就好，Google 大大寫的泥敢嘴？

使用心得

  • 用過 iOS 的 Autolayout 應該都蠻有即視感的。不過 Android 這套比較容易上手，可能也是因為我都在寫 Android 啦。
  • Autoconnect 和 Inference 大多時候很方便，只是工程師個性的我不喜歡，尤其是看到 Autoconnect 的箭頭開始慢慢連過去，我只想叫它快點給我住手啊 (?)。
  • Performance：根據這篇快了約 10% 左右，階層更深的 layout 效果應該會更顯著，而且 hierarchy view 漂亮很多呀…
  • 短短的試用時間就遇到幾次畫面 “out of sync”，那畫面太美我不敢看

基本上這還是 preview 階段，問題應該在正式版本有所改善。以後專案用到的話，大概還是會以手刻 XML 為主，design mode 為輔吧（又多了好多 XML attributes 要背…）。
